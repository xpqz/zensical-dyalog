# PR notes: #27 Implement the caption numbering solution

Branch: `27-implement-the-caption-numbering-solution`
Plan: `docs/plans/zensical-port-plan.md` (Phase 5); decision recorded on #14.

## Approach

Phase 2 (#14) established that a CSS-counter solution is insufficient: the
corpus has eight `[Table N](#_table-N)` cross-references (in
`unix-installation-and-configuration-guide/.../environment-variables.md`), and
without the plugin the `Table:` line renders as a plain paragraph, so there is
no `<caption>` element for CSS to number or anchor. A numbering module is
required.

Zensical cannot run the MkDocs `mkdocs-caption` plugin, but it hands any
extension named in `markdown_extensions` straight to the shared Python-Markdown
parser (verified: `zensical.config._convert_markdown_extensions` has no
whitelist). So the solution is a small Python-Markdown extension,
`tools/dyalog_caption.py`, that reproduces mkdocs-caption's output on the tree.

## The contract (captured from mkdocs-caption 1.3.0)

Built with the corpus's committed config (`start_index: 1`,
`caption_prefix: 'Table {index}:'`, `position: top`), a `Table: <text>`
paragraph before a table produces:

```
<table id="_table-N">
<caption style="caption-side:top">Table N: <text></caption>
...
</table>
```

N is numbered per page from 1, and the `[Table N](#_table-N)` cross-references
resolve to the table `id`. The extension hardcodes these values rather than
exposing options: it exists only to reproduce this one behaviour.

## Open decision (settle in review): build-time import

The unit tests import `dyalog_caption` directly (pytest puts `tools/` on the
path), so they need no packaging. The Zensical build is different: Python-Markdown
imports the extension by name, so the module must be on `sys.path` when
`zensical build` runs from the `zensical/` directory. Two options:

1. Recommended: a minimal installable package so `dyalog_caption` is importable
   everywhere (venv, `check_flatten`, CI), added to the build requirements. No
   env coupling; CI installs it like any dependency.
2. Set `PYTHONPATH=tools` for the build (in `check_flatten.run_build` and CI).
   Lighter, but couples the build invocation to an env var.

I lean to (1). The choice does not affect the extension code or the unit tests,
only how the build resolves it.

## Cycle 2 (GREEN): implementation

### What was implemented

`CaptionTreeprocessor.run` walks each block-level parent, pairs a `Table:`
paragraph with the table element immediately following it, sets
`id="_table-N"` on the table, inserts the `<caption style="caption-side:top">`
as the table's first child, and removes the paragraph. The counter is local to
`run`, so numbering restarts at 1 per document (the reset test). Caption text
folds internal whitespace so a wrapped paragraph renders on one line, which
also settles the review's `DOTALL` Note: the regex matches one paragraph's
text and the fold normalises it.

`merge_configs` appends the `dyalog_caption` extension to the merged
`markdown_extensions` (constant `CAPTION_EXTENSION`), so the emitted
`zensical.toml` carries it.

Build-time import (the open decision): a minimal `tools/pyproject.toml` packages
the extension as the module `dyalog_caption`, installed with
`pip install -e tools/`. Python-Markdown then imports it by name during
`zensical build` regardless of cwd. `requirements-dev.txt` documents the install.

### Implementation review

`docs/reviews/27.md`: Approve, no changes required. All findings are Notes and
are recorded there. Two are worth carrying forward:

- Captions containing inline Markdown (e.g. `Table: The *foo* variable`) would
  truncate at the first inline child, because the treeprocessor reads only the
  paragraph's `.text`. All eight corpus captions are plain text, so it does not
  bite; a follow-up would be needed only if formatted captions are ever
  authored.
- CI (Phase 7, #30) must run `pip install -e tools/` explicitly: the editable
  install is documented in `requirements-dev.txt` but is not an installable
  line, so `pip install -r` alone will not make `dyalog_caption` importable for
  the build.
