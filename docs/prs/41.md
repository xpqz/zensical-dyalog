# PR notes: #41 documentation-assets must be a submodule, not copied files

Branch: `fix-documentation-assets-submodule`
Plan: `docs/plans/zensical-port-plan.md` (Phases 3-4). Corrects the flatten from #20/#21.

## Problem

`convert.py` copies `documentation-assets` into the generated tree as regular
files (93 committed since #21). That freezes the shared corporate style instead
of tracking it: `documentation-assets` is a submodule in the source
(`https://github.com/Dyalog/documentation-assets`) reused across many Dyalog
projects. The output repo must reference it as a submodule and pick up updates
by bumping the submodule commit.

## Fix

- Add the submodule at `zensical/docs/documentation-assets` (where the build
  reads it, so no build-time relocation), pinned to the commit the source uses.
- `convert.py` stops copying `documentation-assets`, and its wipe changes from a
  wholesale `rmtree(zensical)` to regenerating only convert-owned paths
  (content, config, overrides, mathjax) while leaving the submodule checkout
  intact.
- Remove the 93 committed files; add `.gitmodules` + the gitlink.

Consequence for #30 (CI): the pipeline must run `git submodule update --init`
before building. The #30 draft's "no submodule step" was wrong and is reworked
after this lands.

## Cycle 1 (RED): test surface

Two tests in `tools/test_convert.py` capture the new behaviour:
- `test_does_not_copy_documentation_assets`: convert does not create
  `docs/documentation-assets` (it is the submodule's path, not convert's).
- `test_preserves_the_documentation_assets_submodule_on_regen`: a file planted
  at `docs/documentation-assets/...` survives a re-run (the wipe leaves the
  submodule checkout alone).

The old `test_copies_documentation_assets_into_docs` is replaced by the first.

### RED run

`python -m pytest tools/test_convert.py`: 2 failed, 37 passed. The first fails
`AssertionError: assert not True` (convert still copies it in), the second
`AssertionError: 'body {}\n' == 'SUBMODULE SENTINEL\n'` (the wholesale wipe
replaced the planted file). No import or collection errors.

## Anticipated implementation order (Cycle 2)

1. `convert.py`: drop the `documentation-assets` copytree; rework the wipe to
   preserve `output/docs/documentation-assets`; copy the root `docs` with
   `dirs_exist_ok`. Remove the now-dead `.git` exclusion (it existed only for
   the copied submodule) and the associated VCS-metadata test and fixture entry,
   since a real submodule legitimately carries a `.git`.
2. Repo: `git rm` the 93 committed `documentation-assets` files;
   `git submodule add https://github.com/Dyalog/documentation-assets
   zensical/docs/documentation-assets`, pinned to the source's commit.
3. Regenerate `zensical/`; confirm convert leaves the submodule untouched, the
   build resolves the assets from it, and `check_flatten` stays clean.

## Cycle 2 (GREEN): implementation

### What was implemented

- `convert.py`: `copy_content` no longer copies `documentation-assets`; a new
  `_clean_owned` wipes convert-owned paths while keeping `docs/` and the
  `documentation-assets` child intact (the submodule), and the root `docs` copy
  uses `dirs_exist_ok` since `docs/` now persists. The dead `.git` exclusion is
  removed. Module and `copy_content` docstrings updated to the new invariant
  ("owns everything under zensical/ except the documentation-assets submodule").
- `fixture_tree.py`: dropped the `.git` gitlink entry (the VCS test it fed was
  replaced in RED).
- Repo: the 93 copied files are removed; `documentation-assets` is added as a
  submodule (`.gitmodules` + gitlink) at `zensical/docs/documentation-assets`,
  pinned to `c6ad6b7` (the commit the source uses).
